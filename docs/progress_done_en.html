2. Completed modules — detail

01 — Project setup & app foundationDone
  • Flutter project with feature-based folder structure (auth, admin, courses, lessons, batches, student, profile, notifications, progress)
  • Supabase client configured with environment variables (.env)
  • go_router navigation with role-adaptive shell (admin / instructor / student)
  • Light theme + dark theme — user can switch; preference saved locally
  • Arabic + English UI — full manual localization (app_localizations.dart)
  • Responsive layouts (mobile, tablet, desktop) with adaptive scaffolds
  • Shared widgets: loading states, error views, form fields, settings controls
  • Riverpod for state management across the app
02 — Registration, login & user rolesDone
  • Student registration with form validation (localized messages)
  • Email/password login and logout
  • Forgot password flow (email reset link)
  • Email verification handling + “Resend verification email” on login
  • Role-based routing after login: Admin, Instructor, Student dashboards
  • Session persistence (stay logged in)
  • User-friendly error messages (Supabase auth/storage/API mapped to AR/EN)
03 — Admin dashboardDone
  • Dashboard with statistics (users, courses, enrollments)
  • User list with search
  • Change user role (promote to admin / instructor / student)
  • Training centers: create, edit, delete (CRUD)
  • Admin-only navigation and access
04 — Lecturer (instructor) managementDone
  • Instructor role with dedicated dashboard and navigation
  • Profile per instructor (via profiles table)
  • Instructors linked to courses and batches
  • Admin can assign instructor role to any user
05 — Courses, lectures & sessions (full editor)Done
  • Create, edit, delete courses
  • Modules (units) inside each course
  • Lessons inside each module with title, description, order
  • Course editor screen for instructors
  • Course form for metadata (title, description, level, etc.)
  • Instructor course list and course detail views
06 — Learning materials & access controlDone
  • Upload video, PDF, and audio per lesson (Supabase Storage)
  • Row Level Security (RLS) — only enrolled / authorized users access materials
  • Lesson player: video player widget, document viewer (PDF)
  • Lesson editor for attaching materials to lessons
  • Safe storage paths for Arabic/non-ASCII filenames (prevents upload errors)
  • Storage bucket policies in migrations
07 — Student management & enrollmentDone
  • Batches (student groups) per course
  • Roster: add and remove students from a batch
  • Student self-enrollment in published courses
  • “My Courses” on student dashboard
  • Enrollment records in database with RLS
  • Trigger: notification on new enrollment
08 — Course catalog & online bookingDone
  • Student catalog with course cards
  • Search and filter courses
  • Course detail and enroll action
  • Enrollment confirmation flow
  • Note: enrollment is free/self-service — paid checkout not built yet (Module 09)
13 — NotificationsDone
  • In-app notifications list screen
  • Mark notification as read
  • Enrollment alerts (database trigger creates notification)
  • Notifications accessible from app shell
14 — Progress trackingDone
  • Video resume position saved per lesson
  • Auto-complete lesson at 90% watched
  • Course progress bars on student views
  • Progress stored in lesson_progress table with RLS
15 — User profile & avatarDone
  • Profile screen for all roles
  • Edit profile: name, phone, bio, etc.
  • Avatar upload to Supabase Storage
